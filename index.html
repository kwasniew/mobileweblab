Nowoczesny (Mobile) Web Development

Podczas laboratoriów będziemy budować wyszukiwarkę kontaktów z możliwością filtrowania po numerze telefonu i imieniu

Lista kroków (ważne pojęcia)

  1. Hello node.js (node.js, embedded container)
    • Instalacja node.js
    • Tworzymy plik app.js
    • Hello world
    • Omówienie kodu programu
    • node app.js
    • Sprawdzamy localhost
  2. Zwracamy listę kontaktów (HTTP)
    • Dane do zadania
    • Listę kontaktów kopiujemy do kodu programu
    • Zwracamy listę kontaktów zamiast Hello World
    • Przydatne: Content Type: application/json
    • Przydatne: JSON.stringify()
  3. Formularz do wyszukiwania nowych kontaktów (request routing, HTML5 forms)
    • express.js
    • npm - node package manager
    • npm install express
    • express.static()
    • static index.html
    • app.get('/contacts')
    • formularz z polami name (input=text) i phone (input=tel)
  4. Pobieranie wszystkich kontaktów bez filtrowania (DOM, DOM templating, AJAX)
    • jQuery/zepto.js (DOM - zdarzenia, inspekcja, manipulacja + AJAX)
    • $.getJSON()
    • DOM templating
    • Szablon do kontaktów (HTML)
    • Szablon do kontaktów (JS)
    • Wypełniamy szablon danymi z serwera
  5. Filtrowanie kontaktów (Programowanie funkcyjne, REPL)
    • Pobieranie kontaków przy wpisywaniu danych do formularza (on('input'))
    • Wysyłanie query parameters (name i tel) do serwera (val() i req.query.paramName)
    • Filtrowanie danych po stronie serwera z użyciem filter
    • Implementacja filtrowania: REPL driven development
  6. Stylowanie (mobile first front end framework, Chrome Dev Tools, viewport)
    • Testowanie na telefonie lub w emulatorze (Chrome Dev Tools)
    • Ustawienia viewport dla urządzeń mobilnych
    • Stylowanie z bootstrap
    • body class='container'
    • boostrap forms - czytelny formularz
  7. Czyszczenie kodu JS (clean code, module pattern, segregated DOM)
    • selektory DOM w zmiennych
    • duplikacja w budowaniu URL
    • duplikacja w getJSON
    • on filter change
    • on contacts data
    • Single Responsibility Principle
    • osobny plik filter.js
    • module pattern
    • warstwy
